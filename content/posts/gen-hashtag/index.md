게시글 내용을 기준으로 해쉬 태그를 자동으로 생성하고 싶었다  
매번 게시글을 작성 한 뒤 태그를 추가하고 싶은데 너무 귀찮았다  
보통 글을 먼저 작성 한 뒤 제목을 생각하고 적는데, 제목도 귀찮은데 태그를 생각하는 것도 더 귀찮다  
태그를 생각해 내는 과정을 보통 이렇다  

1. 작성한 글을 다시 읽는다
2. 머리속에서 요약을 한다
3. 키워드를 나열한다
4. 키워드에 해당하는 상위 개념 단어를 태그 후보로 삼는다
5. 두 키워드 조합을 섞어서 새로운 키워드를 만든다
6. 키워드를 태그로 게시글 메타 데이터에 추가한다

아... 너무 귀찮은 작업이다  
단순히 글만 작성하고 싶은데, 글을 정리하는 귀찮음이 또 생긴다  
비생산적이다  

비 생산적인 작업은 AI 한테 던지자  
던지기 전에, 잼미니랑 대화를 해봤다  

"블로그 운영 중인데, 해쉬태그 10개만 추출 하는 방법이 없을까 ?"  
"ai 한테 글 전체를 던지고, 이렇게 프롬프트를 입력하세요"  
"오, ㅇㅋ"  

미리 작성된 블로그 게시글 몇개를 던져서 프롬프트를 테스트 해보았다  
결과물이 맘에 안들어서 몇번 프롬프트를 수정하고...  

만족 스런 해쉬태그 생성 프롬프트를 얻었다  

```
글 내용을 요약하는 해시태그 10개를 추출해줘. 글은 조금 있다가 줄게, 우선 방법을 설명해줄게.
1. 문서에 등장하는 단어를 중심으로 주요 단어를 우선 선별
2. 선별된 단어들을 기준으로 핵심 키워드 5개 (명사 위주)
3. 핵심 키워드가 문서에 등장하는 단어가 아니라, 그 단어의 상위 개념의 단어인 경우, 문서에 직접적으로 등장하는 단어로 키워드 5개 (명사 위주)
4. 만약, 해쉬태그 10개가 추출되지 않았으면, 2번과 3번 과정을 반복하여 해시태그 10개 추출

* 형식: #키워드 형태로만 나열해줘.
* 조건1: 해쉬태그 10개 모두를 1개의 json 배열 형태로 답변해줘
* 조건2: pretty json 형태 말고 1줄 json 배열 형태로 써줘
* 조건3: 해쉬태그를 포함한 json 배열 말고 다른 대답은 하지마
```

이제 블로그 글 작성하고 나면  
1. 잼미니 웹 페이지 열고
2. 프롬프트 입력하고
3. 작성한 마크다운 파일 던지고
4. 태그 복붙 하고

파이썬 스크립트로 잼미니 호출하고, 블로그 게시글 수정 한뒤, git push 하는 자동화 과정을 만들어서 git action 에 추가해도 되지만  
뭐 굳이...  
복붙 2번 하면 되는 작업이고  
한번에 여러 게시글을 쓰는 것도 아니고  
작성된 태그를 한 번 보고 수정하는 재미도 있고 (내가 작성한 글이 이런 키워드로 요약되는구나 하고 지켜보는 재미가 있더라)  

일단 이 방법으로 고고
