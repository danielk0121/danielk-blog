## 기록
- SSG
  - 마크다운을 웹으로 변환하는 유명한 툴이 2개 있음
  - 이러한 도구를 정적 사이트 생성기 SSG 라고 함
- 제킬과 휴고 중에 고랭을 사용하고 대량의 파일도 고속으로 처리한다는 장점에 끌려서 휴고를 선택함
- 다른 ssg 툴도 아마 가능하겠지만, 온라인 배포 이전에, 로컬에서 실행해서 웹 블로그를 미리 확인 할 수 있다

## SSG 특징 비교

| 도구명 | 주요 언어 | 장점 및 특징 | 단점 | 추천 대상 |
| :--- | :--- | :--- | :--- | :--- |
| **Hugo** | **Go** | **압도적인 빌드 속도**, 단일 실행 파일 설치, 기능 내장(다국어 등) | Go 템플릿 문법이 다소 생소함 | 빠른 속도를 중시하는 대규모 사이트 |
| **Jekyll** | Ruby | **GitHub Pages** 공식 지원, 방대한 레퍼런스와 커뮤니티 | 페이지가 많아지면 빌드 속도가 느려짐 | GitHub Pages 연동을 최우선하는 입문자 |
| **Hexo** | Node.js | 한국/중국 사용자층 두터움, 블로그 특화 테마 다양 | 플러그인 의존성이 높고 대규모 사이트에 불리 | 예쁜 테마를 찾는 Node.js 사용자 |
| **Astro** | JS/TS | **Zero JS** 기반의 높은 성능, 모든 프레임워크와 혼용 가능 | 비교적 최신 기술이라 학습 자료가 적음 | 최신 스택과 성능 최적화를 원하는 개발자 |
| **Gatsby** | React | React 생태계 활용 가능, 강력한 GraphQL 데이터 관리 | 학습 곡선이 높고 초기 설정이 매우 무거움 | React 숙련자, 복잡한 기능의 웹 앱 스타일 |
